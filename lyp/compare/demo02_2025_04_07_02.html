# 类比编程语言

# 百花齐放、百家争鸣

语言 编译过程 执行过程 用户操作
Java 1. 使用javac将源代码（.java）编译成字节码（.class）。
2. JVM加载字节码文件。		 JVM解释执行字节码，或通过JIT编译器将字节码编译为机器码后执行。 1. 编译：javac HelloWorld.java
2. 运行：java HelloWorld
Rust 1. rustc编译器将源代码转换为汇编代码。
2. 汇编器将汇编代码转换为机器码。
3. 链接器生成可执行文件。		 直接运行生成的可执行文件。直接运行生成的可执行文件。 1. 编译：rustc main.rs （windows编译为exe文件。）
2. 运行：./main（或使用cargo run）
Go 1. 编译器将源代码编译为中间代码。
2. 中间代码优化并生成机器码。
3. 链接生成可执行文件。		 直接运行生成的可执行文件。直接运行生成的可执行文件。 1. 编译：go build
2. 运行：./main（或使用go run直接运行）
Python 1. Python解释器将源代码（.py）编译成字节码（.pyc）。
2. 字节码文件保存在__pycache__目录。		 Python虚拟机逐条解释执行字节码。 运行：python hello_world.py（编译和执行一步完成）
C 1. 预处理器处理宏定义和头文件。
2. 编译器将源代码转换为汇编代码。
3. 汇编器生成目标文件。
4. 链接器生成可执行文件。		 直接运行生成的可执行文件。 1. 编译：gcc hello.c -o hello
2. 运行：./hello
C++ 1. 预处理器处理宏定义和头文件。
2. 编译器将源代码转换为汇编代码。
3. 汇编器生成目标文件。
4. 链接器生成可执行文件。		 直接运行生成的可执行文件 1. 编译：g++ hello.cpp -o hello
2. 运行：./hello
C# 1. 使用csc将源代码（.cs）编译成中间语言（IL）代码。
2. .NET运行时将IL代码即时编译为机器码。		 .NET运行时解释执行或即时编译为机器码后执行。 1. 编译：csc Program.cs
2. 运行：dotnet Program.dll
JavaScript 1. 浏览器或Node.js引擎将源代码解析为抽象语法树（AST）。
2. AST转换为字节码或机器码。		 浏览器或Node.js引擎直接执行代码。 运行：在浏览器中打开HTML文件或使用node script.js运行脚本

这些是本人常用的编程语言总结对比。

# 天下归一

  • 编译型语言（如C、C++、Rust、Go）：编译过程显式，生成可直接运行的机器码或可执行文件。
  • 解释型语言（如Python、JavaScript）：编译过程隐式，运行时动态编译和执行。
  • 中间语言（如Java、C#）：编译成中间字节码，运行时通过虚拟机进一步编译或解释执行。
