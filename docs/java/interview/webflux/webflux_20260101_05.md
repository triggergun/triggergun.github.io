# 流数据是怎么产生的

```java
@GetMapping(
    value = "/users/stream",
    produces = MediaType.TEXT_EVENT_STREAM_VALUE
)
public Flux<User> streamUsers() {

    return Flux.interval(Duration.ofSeconds(1))
            .map(i -> new User(
                    String.valueOf(i),
                    "用户" + i
            ));
}
```

map是对谁遍历？i为会自增？

这是理解 WebFlux 和 Reactor 的关键。

你看到的代码：

```java
Flux
    .interval(Duration.ofSeconds(1))
    .map(index -> {

        User user = new User();

        user.setId(String.valueOf(index));

        user.setName("用户-" + index);

        return user;
    });
```

很多人第一次看都会问：

> map到底遍历谁？
>
> index为什么会自动增长？

------

## 先看 Flux.interval()

实际上：

```java
Flux.interval(Duration.ofSeconds(1))
```

等价于创建了一个无限流：

```
0
1
2
3
4
5
6
...
```

每隔1秒产生一个Long类型数据。

类型实际上是：

```
Flux<Long>
```

即：

```java
Flux<Long> flux =
        Flux.interval(Duration.ofSeconds(1));
```

------

## 它内部在干什么

可以理解成：

```java
while(true){

    Thread.sleep(1000);

    emit(counter++);

}
```

伪代码：

```
0
↓ 1秒

1
↓ 1秒

2
↓ 1秒

3
↓ 1秒

...
```

所以：

```
index
```

实际上就是：

```
0
1
2
3
4
...
```

------

## map是对谁遍历

map遍历的是：

```
Flux.interval()
```

产生的每一个元素。

例如：

```java
Flux.interval(Duration.ofSeconds(1))
```

产生：

```
0
1
2
3
```

然后：

```
.map(index -> ...)
```

相当于：

```
0 -> User(0)
1 -> User(1)
2 -> User(2)
3 -> User(3)
```

流程：

```
Flux<Long>

0
1
2
3

      map

↓
↓
↓

User(0)
User(1)
User(2)
User(3)

Flux<User>
```

------

## 类似Java Stream

你熟悉的话：

```java
List<Integer> list =
    Arrays.asList(1,2,3);

list.stream()
    .map(i -> i * 10)
```

结果：

```
1 -> 10
2 -> 20
3 -> 30
```

Reactor中的：

```java
Flux<Long>
    .map(i -> ...)
```

思想完全一样。

区别只是：

Stream一次处理完。

Flux可以不断产生数据。

------

## 加日志看看

```java
@GetMapping(
        value="/users/stream",
        produces=MediaType.TEXT_EVENT_STREAM_VALUE
)
public Flux<User> streamUsers(){

    return Flux
            .interval(Duration.ofSeconds(1))
            .map(index -> {

                System.out.println(
                        "收到index="+index
                );

                return new User(
                        String.valueOf(index),
                        "用户-"+index
                );
            });
}
```

控制台：

```
收到index=0

收到index=1

收到index=2

收到index=3

收到index=4
```

------

## Reactor中的数据流思想

你可以把它想成管道：

```
Flux.interval()

0
1
2
3
4
5
...
```

进入：

```
.map(...)
```

变成：

```
User-0
User-1
User-2
User-3
...
```

进入：

```
.filter(...)
```

例如：

```java
.filter(user ->
    Long.parseLong(user.getId()) % 2 == 0
)
```

变成：

```
User-0
User-2
User-4
User-6
...
```

进入：

```
.take(5)
```

变成：

```
User-0
User-2
User-4
User-6
User-8
```

然后发送给浏览器。

------

## 更接近实际项目的写法

实际开发中，`index` 很少来自 `interval`。

更多是数据库查询结果：

```java
Flux<User> users =
    userRepository.findAll();
```

假设数据库有：

```
Tom
Jack
Lucy
Bob
```

那么：

```java
users.map(user -> {
    user.setName(
        user.getName().toUpperCase()
    );
    return user;
});
```

实际遍历的是：

```
Tom
Jack
Lucy
Bob
```

转换后：

```
TOM
JACK
LUCY
BOB
```

所以记住一句话：

> `map()` 永远是对 Flux 中流过的每一个元素进行转换。

在你的例子里：

```
Flux.interval()
```

流过的是：

```
0
1
2
3
4
...
```

所以 `index` 自动递增；而 `map(index -> ...)` 就是在把这些数字一个个转换成 `User` 对象。
