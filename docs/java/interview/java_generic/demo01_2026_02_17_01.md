## 泛型基本概念

泛型—— 一种可以接收数据类型的数据类型。

## 没有泛型java程序存的问题

我们都知道，继承是面向对象的三大特性之一，比如在我们向集合中添加元素的过程中add()方法里填入的是Object类，而Object又是所有类的父类，这就产生了一个问题——添加的类型无法做到统一。 由此就可能产生<font color="red">在遍历集合取出元素时类型不统一而报错问题</font>。

![image-20260217191037901](demo01_2026_02_17_01.assets/image-20260217191037901.png)

传统的添加元素的方式(没有泛型的时候)不能对加入到集合ArrayList中的<font color="red">数据类型进行约束</font>(不安全)。遍历的时候，需要进行类型转换。如果集合中的数据量较大，对<font color="red">效率有影响</font>。 这就极大地<font color="red">降低了程序的健壮性</font>，因此设计者针对此问题引入了泛型！
