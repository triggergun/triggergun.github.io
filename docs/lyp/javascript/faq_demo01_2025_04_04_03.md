# 为什么 Vue、React、Nuxt 都用 ESM？

因为打包工具喜欢 ESM。

例如：

```js
import { ref } from 'vue'
```

如果你没用：

```js
computed
watch
reactive
```

打包器可以直接删除。

最终：

```js
包更小
速度更快
```

这叫：

```js
Tree Shaking
```

CommonJS 很难做到。

------

## Node.js 现在是什么情况

现代 Node.js 两种都支持。

------

**CommonJS模式**

package.json

```js
{
  "type": "commonjs"
}
```

或者省略：

```js
{}
```

默认就是：

```js
require()
module.exports
```

------

**ESM模式**

package.json

```js
{
  "type": "module"
}
```

然后：

```js
import fs from 'fs'
export default {}
```

都能直接使用。

------

## 实际开发怎么选

**Vue/Nuxt/Vite**

用：

```js
export default
import
```

即：

```js
ESM
```

------

**React/Next**

用：

```js
export default
import
```

即：

```js
ESM
```

------

**Node.js 新项目**

推荐：

```js
{
  "type": "module"
}
```

使用：

```js
import
export
```

即：

```js
ESM
```

------

**老 Node.js 项目**

很多还在：

```js
require()
module.exports
```

即：

```js
CommonJS
```

------

## 一句话记忆

```js
CommonJS = Node.js 早期自创模块规范
           require + module.exports

ESM       = JavaScript 官方模块规范
           import + export

现在前端几乎全部使用 ESM，
Node.js 新项目也推荐 ESM，
CommonJS 主要用于兼容历史项目。
```

所以你看到：

```js
export default {}
```

基本可以直接判断：

```js
这是 ESM 模块写法
常见于：
✓ Vue
✓ Nuxt
✓ React
✓ Next
✓ Vite
✓ 现代 Node.js

不是传统 CommonJS 写法
```
